# 3.1 Data Pipeline

## Dependencies
**Requires:** 2.1 TIPS Basics (for yield calculations)

**This file specifies the data acquisition and storage infrastructure.**
The application runs entirely client-side using static CSV files hosted on Cloudflare R2. These files are updated daily/monthly by Node.js scripts running in GitHub Actions.

---

## Architecture

Data flows from Treasury sources to Cloudflare R2 via GitHub Actions. The web application reads exclusively from R2.

## Data Sources

| Script | Source | Frequency | Output |
|---|---|---|---|
| `getTipsYields.js` | FedInvest Price Detail | Daily (~1 PM ET) | `TipsYields.csv` |
| `fetchTipsRef.js` | FiscalData Auctions | On change | `TipsRef.csv` |
| `fetchRefCpi.js` | TreasuryDirect SecIndex | Monthly (mid-month) | `RefCPI.csv` |

---

## Environment Variables

Scripts use the `@aws-sdk/client-s3` library to upload directly to Cloudflare R2. Requires the following environment variables in CI/CD (GitHub Secrets):

| Variable | Description |
|---|---|
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare Account ID (from URL or Dashboard home) |
| `R2_BUCKET` | Target bucket name (e.g., `tips-ladder`) |
| `R2_ACCESS_KEY_ID` | R2 S3 API Token Access Key |
| `R2_SECRET_ACCESS_KEY` | R2 S3 API Token Secret Key |

### Generating R2 Tokens

**Method A: R2 Dashboard (Recommended)**
1. Cloudflare Dashboard → R2 → "Manage R2 API Tokens" → "Create API token".
2. Permissions: **Object Read & Write**.
3. TTL: Forever.

**Method B: Custom Account Token**
1. Manage Account → API Tokens → Create Token → Custom Token.
2. Permissions: **Account** · **Workers R2 Storage** · **Edit**.

## File Specifications

### 1. TipsYields.csv
Primary data file for the application. Contains today's snapshot of the TIPS market.

**Columns:**
- `settlementDate`: Date prices were captured (YYYY-MM-DD)
- `cusip`: 9-character ID
- `maturity`: Maturity date (YYYY-MM-DD)
- `coupon`: Annual coupon rate (decimal, e.g., 0.00125)
- `baseCpi`: Reference CPI on dated date
- `price`: Clean price (percentage of par). Source priority: Buy > Sell > EOD.
- `yield`: Calculated YTM (decimal).

**Yield Calculation:**
Uses `yieldFromPrice` (actual/actual convention) matching Excel's `YIELD` function.

### 2. TipsRef.csv
Metadata reference for all outstanding TIPS (including historical). Used by `getTipsYields.js` to look up coupons and base CPIs for the CUSIPs found on FedInvest.

**Columns:**
- `cusip`
- `maturity`
- `datedDate`
- `coupon`
- `baseCpi`
- `term` (e.g., "5-Year", "10-Year")

### 3. RefCPI.csv
Daily reference CPI values for the current month and history. Used by the web app for calculating `indexRatio`.

**Columns:**
- `date`: YYYY-MM-DD
- `refCpi`: Number (e.g., 308.417)

## Workflows

### `get-tips-yields.yml`
- **Schedule:** Mon-Fri at 17:00 UTC / 18:00 UTC (1:00 PM ET)
- **Logic:** Fetches `TipsRef.csv` from R2 first (to ensure metadata is available), then scrapes prices, calculates yields, and uploads `TipsYields.csv`.

### `fetch-ref-cpi.yml`
- **Schedule:** Monthly (mid-month) to capture new CPI releases.
